# catPlan AI 功能集成调研

## 调研背景

catPlan 是微信小程序任务管理系统，希望集成 AI 功能提升用户体验。本次调研基于阿里云百炼平台，优先使用 Qwen3.5-Plus 和 Qwen3-Max 模型。

### 约束条件
- **必须支持**: 阿里云百炼 Coding Plan
- **优先模型**: Qwen3.5-Plus、Qwen3-Max
- **环境限制**: 微信小程序（需通过后端服务器中转调用 API）

---

## AI 功能一：AI 任务助手（智能任务建议）

### 功能描述
根据用户历史任务、当前进度和时间安排，智能生成个性化任务建议和优先级排序。

### 使用场景
- 用户创建新任务时，AI 推荐相关子任务
- 每日/每周任务规划建议
- 任务优先级智能排序
- 相似任务模板推荐

### 实现方案
```
用户输入 → 微信小程序 → 后端服务器 → 阿里云百炼 API → 返回建议 → 小程序展示
```
- 使用 Qwen3.5-Plus 进行任务分析和建议生成
- 需要维护用户任务历史上下文（最近 10-20 条任务）
- Prompt 设计：包含用户角色、历史任务模式、当前目标

### API 成本
- **模型**: Qwen3.5-Plus
- **输入**: 约 500 tokens（任务历史 + 上下文）
- **输出**: 约 300 tokens（建议内容）
- **单次成本**: 约 0.002~0.005 元/次
- **日均调用**: 假设 1000 用户 × 2 次/天 = 2000 次
- **日成本**: 4~10 元/天

### 开发工时
- 后端 API 集成：2 天
- Prompt 工程优化：2 天
- 小程序 UI 适配：1 天
- 测试调优：1 天
- **合计**: 6 人天

### 用户价值
- ⭐⭐⭐⭐⭐ 高频使用场景
- 显著提升任务管理效率
- 增强用户粘性

### 推荐指数
**★★★★★ (9/10)** - 核心功能，强烈推荐

## AI 功能二：AI 审批助手（智能审批建议）

### 功能描述
对任务完成情况进行智能审核，给出通过/驳回建议及理由。

### 使用场景
- 团队成员提交任务成果后，AI 预审
- 任务完成质量评估
- 提交内容与任务描述匹配度检查
- 风险任务标记（延期、质量异常）

### 实现方案
```
任务描述 + 提交内容 → 后端 → 百炼 API → 审批建议 + 理由 → 管理员确认
```
- 使用 Qwen3-Max 进行深度分析和判断
- 需要任务描述、提交内容、验收标准作为输入
- 输出结构化：{建议：通过/驳回/需补充，理由：xxx，风险点：[]}

### API 成本
- **模型**: Qwen3-Max（更高精度）
- **输入**: 约 800 tokens（任务描述 + 提交内容）
- **输出**: 约 200 tokens（审批意见）
- **单次成本**: 约 0.005~0.01 元/次
- **日均调用**: 假设 500 次审批/天
- **日成本**: 2.5~5 元/天

### 开发工时
- 后端 API 集成：2 天
- 审批规则设计：2 天
- 结构化输出处理：1 天
- 管理员确认流程：1 天
- **合计**: 6 人天

### 用户价值
- ⭐⭐⭐⭐ 提升审批效率
- 减少人工审核工作量
- 降低主观判断偏差

### 推荐指数
**★★★★☆ (8/10)** - 高价值功能，推荐

## AI 功能三：AI 积分评估（智能任务难度评估）

### 功能描述
根据任务描述自动评估任务难度，建议合理积分/工时。

### 使用场景
- 创建任务时自动建议积分
- 任务难度横向对比（避免积分不公）
- 历史任务积分校准参考
- 团队积分体系标准化

### 实现方案
```
任务描述 + 历史相似任务 → 后端 → 百炼 API → 难度等级 + 建议积分
```
- 使用 Qwen3.5-Plus 进行任务复杂度分析
- 参考维度：工作量、技能要求、风险程度、依赖关系
- 输出：难度等级 (1-5) + 建议积分范围

### API 成本
- **模型**: Qwen3.5-Plus
- **输入**: 约 400 tokens
- **输出**: 约 100 tokens
- **单次成本**: 约 0.001~0.003 元/次
- **日均调用**: 假设 300 次/天
- **日成本**: 0.3~0.9 元/天

### 开发工时
- 评估维度设计：2 天
- API 集成：1 天
- 历史数据校准：2 天
- **合计**: 5 人天

### 用户价值
- ⭐⭐⭐⭐ 提升积分体系公平性
- 减少任务分配争议
- 帮助新人理解任务难度

### 推荐指数
**★★★★☆ (8/10)** - 实用功能，推荐

## AI 功能四：AI 客服（智能问答）

- **输出**: 约 200 tokens
- **单次成本**: 约 0.002~0.004 元/次
- **日均调用**: 假设 200 次/天
- **日成本**: 0.4~0.8 元/天

### 开发工时
- 知识库构建：3 天
- RAG 系统搭建：3 天
- 对话管理：2 天
- API 集成：1 天
- **合计**: 9 人天

### 用户价值
- ⭐⭐⭐⭐ 降低客服成本

- **单次成本**: 约 0.005~0.01 元/次
- **日均调用**: 假设 100 次/天（周报月报）
- **日成本**: 0.5~1 元/天

### 开发工时
- 数据统计模块：2 天
- 报告模板设计：2 天
- AI 解读集成：2 天
- 可视化适配：2 天
- **合计**: 8 人天

### 用户价值
- ⭐⭐⭐⭐ 管理者高价值功能

## 技术架构建议

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   微信小程序    │────▶│   后端服务器    │────▶│  阿里云百炼 API │
│  (前端展示)     │◀────│  (API 中转 + 业务)│◀────│  (Qwen 模型)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                              │
                              ▼
                       ┌─────────────────┐
                       │   业务数据库    │
                       │ (任务 + 用户数据)│
                       └─────────────────┘
```

### 关键注意事项
1. **微信小程序限制**：必须在后端服务器中转 API 调用，不能前端直连
2. **用户隐私**：任务数据脱敏后再发送给 AI，避免敏感信息泄露
3. **成本控制**：设置 API 调用限额和告警，避免异常调用
4. **响应速度**：AI 调用设置超时（建议 5-10 秒），超时降级处理
5. **内容安全**：对 AI 输出内容进行审核过滤，避免不当内容

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| API 成本超预算 | 中 | 设置调用限额、告警，优化 Prompt 减少 token |
| AI 响应慢 | 中 | 设置超时降级，异步处理长任务 |
| 输出质量不稳定 | 中 | 持续优化 Prompt，建立反馈机制 |
| 用户隐私泄露 | 高 | 数据脱敏，不传敏感信息，签署保密协议 |
| 微信小程序审核 | 低 | 提前了解 AI 功能审核规则，合规设计 |

基于阿里云百炼平台为 catPlan 集成 AI 功能，**建议优先实施 AI 任务助手和 AI 审批助手**，这两个功能用户价值最高、技术风险最低、ROI 最明显。

**预计投入**：
- 开发成本：约 38 人天（2 人团队 2-3 个月）
- API 成本：约 300-600 元/月（初期），随用户增长增加
- 总预算建议：开发 +6 个月运营 ≈ 10-15 万元

**预期收益**：
- 用户活跃度提升 20-30%
- 任务管理效率提升 30-50%
- 客服成本降低 40-60%
- 产品差异化竞争优势
